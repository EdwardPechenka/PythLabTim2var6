# Инструкция по запуску Лабораторной работы №2 (Вариант 6)

Этот проект содержит модульное программное обеспечение на языке Python для чтения числовых данных из файла, их обработки с помощью отдельного модуля и сохранения финального результата в текстовый отчет. Программа оснащена надежной системой валидации входных данных и обработки исключений (try-except).

## Шаг 1. Размещение файлов и структура проекта
Убедись, что структура папок и файлов в твоем рабочем пространстве точно совпадает с системными требованиями (как в VS Code):

```text
📁 PYTHLABTIM2VAR6/
│
├── 📁 __pycache__/      # Создается автоматически после запуска
├── 📄 get_sum.py        # Модуль с функцией подсчета суммы
├── 📄 main.py           # Главный скрипт управления программой
├── 📄 numbers.txt       # Входной файл с числами
└── 📄 result.txt        # Выходной файл с результатом работы
```

Сценарии для проверки (валидация и тесты)
Тест 1. Успешное выполнение задания:
Если файл numbers.txt содержит корректные данные (например, 10 20.5 30 40.2 50 100.3), программа считает их, передаст в функцию get_sum из импортированного модуля, выполнит расчет и выведет сообщение:
Консоль: Sum (251.0) has written in file result.txt
Результат: В папке автоматически создастся (или обновится) файл result.txt с текстом: Total result: 251.0.

Тест 2. Проверка защиты от отсутствия файла (FileNotFoundError):
Временно переименуй или удали файл numbers.txt и запусти программу еще раз.
Результат: Благодаря обработчику исключений программа не упадет в критическую ошибку, а выведет понятное уведомление: Error, file numbers.txt was not found.

Тест 3. Проверка на некорректные данные (ValueError):
Запиши в файл numbers.txt строку, содержащую буквы или посторонние символы, например: 10 20.5 текст 40. Запусти приложение.
Результат: Скрипт зафиксирует невозможность конвертации букв в дробное число и выведет: Error: the file has incorrect data. Details: could not convert string to float: 'текст'.

Частые ошибки и их решение
❌ Ошибка: "'python' не является внутренней или внешней командой..."

Причина: При установке Python на компьютер не была поставлена галочка "Add Python to PATH" (Добавить Python в переменные среды).

Решение: Попробуй в консоли вместо python написать py main.py. Если не помогает, перезапусти установщик Python, нажми Modify (Изменить) и обязательно отметь флажок Add Python to PATH.

❌ Ошибка: ModuleNotFoundError: No module named 'get_sum'

Причина: Файл main.py не видит модуль get_sum.py, так как они находятся в разных директориях, либо командная строка запущена из другой папки.

Решение: Убедись, что оба файла лежат строго в одной папке PYTHLABTIM2VAR6 и в консоли ты находишься именно в ней.
